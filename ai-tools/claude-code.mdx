Claude Code 是 Anthropic 的官方 CLI 工具。本指南将帮助您设置 Claude Code 来协助编写和维护您的文档。

## 先决条件

- 有效的 Claude 订阅（Pro、Max 或 API 访问权限）

## 设置

1. 全局安装 Claude Code：

  ```bash
  npm install -g @anthropic-ai/claude-code
```

2. 导航到您的文档目录。
3. （可选）将下面的 `CLAUDE.md` 文件添加到您的项目中。
4. 运行 `claude` 开始。

## 创建 `CLAUDE.md`

在文档仓库的根目录创建一个 `CLAUDE.md` 文件，以根据您特定的文档标准训练 Claude Code：

````markdown
# Mintlify 文档

## 工作关系
- 您可以对想法提出异议 - 这可以带来更好的文档。这样做时请引用来源并解释您的理由
- 始终要求澄清而不是做出假设
- 绝不撒谎、猜测或编造信息

## 项目上下文
- 格式：带有 YAML frontmatter 的 MDX 文件
- 配置：用于导航、主题、设置的 docs.json
- 组件：Mintlify 组件

## 内容策略
- 记录足够用户成功的信息 - 不多不少
- 优先考虑信息的准确性和可用性
- 尽可能使内容保持常青
- 在添加新内容之前搜索现有信息。避免重复，除非出于战略原因
- 检查现有模式以确保一致性
- 从进行最小的合理更改开始

## 页面的 Frontmatter 要求
- title：清晰、描述性的页面标题
- description：用于 SEO/导航的简洁摘要

## 写作标准
- 第二人称语气（"您"）
- 程序性内容开头列出先决条件
- 发布前测试所有代码示例
- 匹配现有页面的样式和格式
- 包含基本和高级用例
- 所有代码块都带有语言标签
- 所有图像都包含替代文本
- 内部链接使用相对路径

## Git 工作流程
- 提交时绝不使用 --no-verify
- 开始前询问如何处理未提交的更改
- 当没有明确的分支用于更改时创建新分支
- 在整个开发过程中频繁提交
- 绝不跳过或禁用预提交钩子

## 不要
- 在任何 MDX 文件中跳过 frontmatter
- 对内部链接使用绝对 URL
- 包含未经测试的代码示例
- 做出假设 - 始终要求澄清
````
