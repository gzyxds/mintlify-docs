配置 Windsurf 的 Cascade AI 助手来帮助您编写和维护文档。本指南展示了如何专门为您的 Mintlify 文档工作流程设置 Windsurf。

## 先决条件

- 已安装 Windsurf 编辑器
- 访问您的文档仓库

## 工作区规则

创建工作区规则，为 Windsurf 提供有关您的文档项目和标准的上下文。

在您的项目根目录中创建 `.windsurf/rules.md`：

````markdown
# Mintlify 技术写作规则

## 项目上下文

- 这是一个在 Mintlify 平台上的文档项目
- 我们使用带有 YAML frontmatter 的 MDX 文件  
- 导航在 `docs.json` 中配置
- 我们遵循技术写作最佳实践

## 写作标准

- 在说明中使用第二人称（"您"）
- 使用主动语态和现在时写作
- 从先决条件开始程序
- 为主要步骤包含预期结果
- 使用描述性、关键词丰富的标题
- 保持句子简洁但信息丰富

## 必需的页面结构

每个页面必须以 frontmatter 开头：

```yaml
---
title: "清晰、具体的标题"
description: "用于 SEO 和导航的简洁描述"
---
```

## Mintlify 组件

### 标注

- `<Note>` 用于有用的补充信息
- `<Warning>` 用于重要注意事项和重大变更
- `<Tip>` 用于最佳实践和专家建议  
- `<Info>` 用于中性上下文信息
- `<Check>` 用于成功确认

### 代码示例

- 在适当时，包含完整、可运行的示例
- 对多语言示例使用 `<CodeGroup>`
- 在所有代码块上指定语言标签
- 包含真实数据，而非占位符
- 对 API 文档使用 `<RequestExample>` 和 `<ResponseExample>`

### 程序

- 对顺序说明使用 `<Steps>` 组件
- 在相关时包含带有 `<Check>` 组件的验证步骤
- 将复杂程序分解为较小的步骤

### 内容组织

- 对平台特定内容使用 `<Tabs>`
- 对渐进式披露使用 `<Accordion>`
- 对突出显示内容使用 `<Card>` 和 `<CardGroup>`
- 将图像包装在带有描述性替代文本的 `<Frame>` 组件中

## API 文档要求

- 使用 `<ParamField>` 记录所有参数 
- 使用 `<ResponseField>` 显示响应结构
- 包含成功和错误示例
- 对嵌套对象属性使用 `<Expandable>`
- 始终包含认证示例

## 质量标准

- 发布前测试所有代码示例
- 对内部链接使用相对路径
- 为所有图像包含替代文本
- 确保正确的标题层次结构（从 h2 开始）
- 检查现有模式以确保一致性
````
